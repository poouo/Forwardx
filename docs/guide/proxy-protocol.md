# PROXY Protocol

PROXY Protocol 用来把用户真实 IP 传给后端服务。

普通转发时，后端看到的可能是入口机或出口机 IP。开启并正确配置 PROXY Protocol 后，后端可以拿到用户真实 IP。

## 注意事项

- 主要用于 TCP。
- UDP 不适用。
- 后端服务必须支持 PROXY Protocol。
- 不支持的转发方式或场景，面板会禁用开关。

## 配置含义

PROXY Protocol 现在在端口转发、隧道或转发链的高级设置中配置。规则引用这些资源后，会继承对应资源的配置。

| 配置 | 含义 |
| --- | --- |
| accept proxy | 接收上一跳传来的真实 IP 信息 |
| send proxy | 把真实 IP 信息发送给下一跳 |

如果是：

```text
用户 -> 中转 -> 落地 -> 后端
```

通常需要中转发送，落地接收并继续发送给后端。具体是否开启，要看你的后端服务是否支持 PROXY Protocol。

## 常见误区

- 只在入口开启不一定能让后端看到真实 IP。
- 后端不支持 PROXY Protocol 时，开启后可能导致服务无法识别连接。
- UDP 流量不会通过 PROXY Protocol 传递真实 IP。
- 多级链路中，每一跳是否接收和发送要按实际路径配置。
