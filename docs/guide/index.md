# 教程概览

ForwardX 是一个面向多台 Linux 服务器的转发管理面板。它可以统一管理主机、端口转发资源、隧道链路、端口转发链、转发组、用户权限、流量统计和延迟探测。

2. [部署前准备](./preparation.md)
3. [部署面板](./deploy-panel.md)

## ForwardX 能做什么

- 把多台服务器统一管理起来。
- 给不同用户分配不同转发规则。
- 创建 TCP、UDP 或 TCP+UDP 转发规则，并绑定端口转发、隧道、转发链或转发组资源。
- 使用隧道把入口机器和出口机器连接起来。
- 使用端口转发链实现多跳转发。
- 使用转发组、入口组和出口组做高可用。
- 查看规则是否在线、流量是否增长、链路延迟是否正常。
- 管理用户、套餐、余额、到期提醒和 Telegram 通知。

## 推荐使用路径

```text
部署面板 -> 初始化数据库和管理员 -> 创建 Agent Token -> 安装 Agent -> 创建端口转发或链路资源 -> 添加转发规则 -> 链路测试
```

## 常用页面

| 页面 | 用途 |
| --- | --- |
| 主机管理 | 查看 Agent 主机在线状态、资源占用、IP 地址和版本，管理 Agent Token |
| 转发规则 | 选择端口转发、隧道、转发链或转发组后创建用户实际访问的业务入口 |
| 链路管理 | 管理隧道、端口转发链、转发组、入口组和出口组 |
| 服务管理 | 管理延迟探测服务和图表 |
| 系统设置 | 配置面板公开地址、Telegram、邮件、升级和备份 |
| 用户管理 | 管理用户、权限、额度和到期时间 |
