# 端口转发链

端口转发链用于多台机器按顺序转发。

常见路径：

```text
用户 -> 入口机 -> 中转机 -> 出口机 -> 目标服务
```

进入：

```text
链路管理 -> 端口转发链
```

## 使用建议

- 端口转发链在「链路管理」中单独创建，和隧道链路、普通端口转发分开管理。
- 如果已经选择入口组，下方链路成员不需要再重复添加入口。
- 如果使用入口组和出口组，中间添加的主机就是中转主机。
- 添加链路成员时，注意入口、中转、出口顺序。
- IPv6 转发需要确认链路上的机器都具备对应 IPv6 能力。
- PROXY Protocol、倍率等链路级参数在端口转发链的高级设置中配置，默认收起。
- 端口转发链列表支持拖动排序。

创建转发规则时选择这条端口转发链即可。

## 链路测试

端口转发链创建后，建议使用链路测试检查每一段是否连通。

如果某一段失败，优先检查：

- 当前节点是否在线。
- 当前节点到下一跳是否可访问。
- 入口端口和目标端口是否一致。
- IPv6 开关是否和实际地址类型一致。
- 安全组和防火墙是否放行。
