# Changelog

## [2.3.246] - 2026-07-20

### 新增

- 在线迁移新增“关键数据迁移”和“全量迁移”两种方式；关键迁移跳过监控、延迟、测试及审计历史，全量迁移在两端均为 SQLite 且目标库为空时自动使用数据库文件快速传输。
- SQLite 全量迁移新增流式压缩传输、文件大小与 SHA-256 校验、数据库完整性检查和目标面板接管确认；迁移失败时保留旧面板数据，不会提前切换 Agent。

### 优化

- 结构化迁移改为分批查重、批量写入和事务处理，3000 条规则场景不再逐行查询与提交；迁移进度按实际阶段更新，关联修复和累计流量数据在导入后继续保留。
- 规则延迟统一为真实转发路径口径：直连规则由入口主机探测，隧道规则由标准出口探测目标并叠加当前隧道段延迟；手动测试完成后会立即刷新图表和统计缓存。
- Docker 定时任务增加防重入、错峰启动和历史表时间索引，Agent 流量上报改为批量读取规则、隧道与转发组上下文，减少规则较多时的重复数据库查询。
- Docker 默认启用本地日志轮转，可通过 `FORWARDX_LOG_MAX_SIZE` 和 `FORWARDX_LOG_MAX_FILES` 调整单文件大小与保留数量。
- Agent 旧端口清理按 TCP/UDP 通道执行，Realm、GOST 和 Socat 可继续在同一数字端口使用互不冲突的协议，不会因清理另一协议而中断。

### 修复

- 修复在线迁移长时间停在 20% 或数小时仍未完成的问题；大体量 SQLite 数据不再经过逐表 JSON 展开，全量迁移速度和内存占用明显改善。
- 修复 Docker 部署下清理、统计和探测任务重叠执行，以及缺少时间索引和逐规则查询造成面板进程 CPU、数据库与日志占用持续偏高的问题。
- 修复延迟测试结果、规则卡片和历史图表取值来源不同的问题；清除旧版隧道规则的出口直连样本，避免升级后继续混入错误数据。
- 修复规则更换端口或转发工具后，旧端口删除动作可能被较新的应用动作判为过期而丢弃的问题。
- 修复 UDP 端口占用未被旧清理逻辑发现，以及 Alpine/OpenRC 下 Realm 服务停止后残留子进程仍占用端口的问题。
- 修复 Realm 与 GOST 交接失败时共享运行时持续重启并推高 CPU；新配置必须确认真实监听成功，否则自动恢复上一份可用配置，systemd 连续启动失败也会受到次数限制。

### 验证

- 服务端与相关前端单元测试 `183/183` 通过，在线迁移覆盖 3000 条规则批量导入、部分无效数据隔离、关键数据裁剪及 SQLite 快速迁移。
- Agent 完整 Go 测试、`go vet` 和 3000 条规则压力测试通过；压力场景覆盖动作积压、异常状态、旧端口清理以及 Realm/GOST 协议通道交接；ForwardX FXP 完整测试与 `go vet` 通过。
- `pnpm check:versions`、`pnpm exec tsc --noEmit`、`pnpm build`、`pnpm docs:build` 和 `git diff --check` 通过。

### 版本

- 面板版本升级至 `2.3.246`，APK Release 版本升级至 `2.3.246`。
- Agent 版本升级至 `2.2.162`，ForwardX FXP runtime 日志版本保持 `2.2.105`，Android APP 版本保持 `2.3.94`。

## [2.3.245] - 2026-07-20

### 新增

- 新增配置审计与单调递增修订号，记录主机、隧道和转发规则的创建、修改、删除及期望状态下发；敏感字段递归脱敏，并由 Agent 回报最后接收和应用的修订号与配置摘要。
- 系统日志新增一键生成脱敏支持包，可并发收集在线 Agent 的服务、监听、路由、Mimic、FXP 和近期日志；离线主机不会阻塞任务，单项和单主机输出均有限额。
- Agent 新增主机启动 ID、进程启动时间和恢复进度上报，面板可区分主机重启、Agent 进程重启与普通重连，并展示规则恢复完成度。
- 赞助信息新增 `USDT (Polygon)` 网络及收款地址。

### 优化

- GOST 与 Nginx 共享运行时改为分阶段校验、原子替换和失败回滚；Nginx 证书、私钥与主配置作为同一事务应用，私钥保持 `0600` 权限，校验或重载失败时恢复上一份可用配置和服务。
- Mimic 在实际启用 UDP 混淆时校验命令与内核模块环境，分别上报连接状态和 XDP/TC Hook；空闲连接保持健康。配置删除需经过至少三次完整空快照且持续 60 秒，首次接管失败会恢复原配置及原服务启用/运行状态。
- Mimic 安装脚本默认使用 `wg-mimic-fabric v1.4.9` 安装 `mimic v0.7.1`，支持面板 GitHub 加速配置并保留显式版本覆盖能力；未启用 Mimic 时不会触发环境安装或配置。
- Agent 心跳、插件任务、批量规则和独立资源操作继续按主机或资源键并发处理，降低大量规则恢复、批量下发和插件读取时的串行排队；同一资源仍保持必要的顺序一致性。
- FXP 在停止入口、出口和中转监听后为现有 TCP 会话保留最长 5 秒排空时间，Agent 仅在 6 秒后仍未退出时强制终止；端点异常和恢复按状态变化上报并记录持续时间。
- 批量管理支持端口转发导入、分块并发创建/更新、真正的批量删除、冲突跳过或换端口，并在打开管理窗口时刷新完整规则集合；跨页和部分失败会保留逐项结果。
- 面板备份恢复新增防重复指纹、部分成功统计和明确警告，关联修复、状态重置等收尾失败不会掩盖已导入数据；恢复后会主动唤醒 Agent 并重新检测连接与 Mimic 环境。
- 发布工作流增加面板、Agent、FXP 的类型检查、单元测试和静态检查；Android Release 优先使用仓库 Secrets 提供的正式签名，尚未配置时沿用 community 签名，保证现有社区 APK 可以覆盖升级。

### 修复

- 修复支付跳转和回调地址可能引用当前请求中的错误主机或代理地址；订单创建、到账发放、余额入账和优惠码占用改为数据库事务，失败、取消或过期时可靠释放优惠码。
- 修复同一主机端口在隧道、端口转发、转发组和转发链之间切换时，旧运行时未完成交接便判定端口占用的问题；Agent 先清理旧类型再应用新规则，并按端口统一运行身份。
- 修复批量删除未覆盖刚导入规则、批量导入缺少端口转发目标、单条失败中断整批操作以及重复提交备份产生重复数据的问题。
- 修复纯 UDP 规则使用 TCPing 导致持续超时；纯 UDP 改用 Ping，TCP 或 TCP + UDP 继续使用 TCPing，并拒绝协议类型不匹配的旧探测结果。
- 修复多入口、多中转故障转移和多出口隧道只要单个节点状态异常就把整条链路标红，以及旧拓扑结果混入新配置的问题；存在独立新鲜探测时由探测决定可用性，否则按当前有效主机拓扑判断。
- 修复 Agent 或面板升级、主机重启和大量规则并发恢复时可能缓慢排队、旧端口残留、运行状态长期处于下发中，以及单个 FXP 出口异常误判整条规则不可用的问题。
- 修复 Mimic 首次接管已有配置后启动失败会删除原配置、Nginx 证书已替换但主配置回滚造成不匹配，以及支持包在大量主机下体积过大的问题。
- 规则从 `TCP + UDP` 改为单协议时增加确认；跨转发类型修改仍只保留基础目标地址、端口和协议等共同字段，不支持的高级参数自动忽略。
- 插件 Agent 脚本执行纳入信任范围校验，未信任插件不能执行系统脚本；Agent 任务保留结构化业务错误、分动作状态和最后一次成功列表，失败刷新不再清空整个资源页面。
- 修复 Linux 发布环境中 FXP UDP 直连与中转回环测试的启动竞态：UDP `Dial` 不再被误认为服务端已经监听，首次完整往返遇到入口 socket 尚未就绪产生的 `ECONNREFUSED` 或短暂超时时会在 3 秒上限内重试，避免业务链路正常但 Agent 发布工作流被偶发测试失败阻断。
- 修复仓库尚未配置 Android 正式签名 Secrets 时 Release APK 直接失败的问题；恢复与既有安装包一致的 community 签名兜底，避免改用随机 Debug 签名导致用户无法覆盖升级。

### 验证

- 服务端与相关前端单元测试 `172/172` 通过；Mimic、探测下发和旧数据迁移专项测试 `8/8` 通过。
- `pnpm check:versions`、`pnpm exec tsc --noEmit`、`pnpm build` 和 `pnpm docs:build` 通过。
- Agent 与 ForwardX FXP 的完整 Go 测试、`go vet`、Windows 构建和 Linux amd64 交叉构建通过；本地面板 tRPC 健康检查返回 `status: ok`。
- FXP UDP 直连与中转回环测试连续运行 `50` 轮通过。

### 版本

- 面板版本升级至 `2.3.245`，APK Release 版本升级至 `2.3.245`。
- Agent 版本为 `2.2.161`，ForwardX FXP runtime 日志版本为 `2.2.105`，Android APP 版本为 `2.3.94`。

## [2.3.243] - 2026-07-19

### 新增

- 隧道链路新增“链路中转”和“故障转移”模式；配置至少两台中转主机后，可按列表优先级使用中转节点，主节点不可达时自动切换，恢复后优先回到主节点。GOST 与 ForwardX 均支持该模式，入口和各中转路径会独立探测并汇总状态。
- 出口组新增“不使用、主备、轮询、随机、来源 IP 哈希”策略，并由关联隧道自动继承；策略统一接入 GOST、ForwardX 和 Nginx，TCP 与 UDP 会话保持稳定出口。“不使用”时仅使用优先级最高的第一台已启用主机。
- 出口组成员、顺序或地址变化后会自动同步关联隧道的主出口、额外出口、中转末端、Mimic 端口及规则出口映射；已有主机尽量复用原监听端口，新增主机才分配新端口。

### 优化

- 关闭“允许多设备在线”后不再使其他设备的登录凭据失效：同一时间仍只允许一台设备操作，另一台会进入等待页，并在当前设备停止活动后自动恢复原登录状态，无需重新输入账号密码。
- 隧道主机链路编辑器缩短顶部布局，主机数量紧跟标题展示；中转节点改为实时拖动排序，拖动位置、固定出口和故障转移优先级显示保持一致。
- 中国区域白名单插件升级至 `0.6.2`，新增全部端口、指定单端口或端口范围、高级多端口策略三种作用范围；动态配置改用 `jq` 解析并移除 Python 依赖，iptables 与 nftables 均可只配置端口白名单而不误套用为全局规则。
- Agent 执行插件任务前会检查清单声明的解释器是否存在，缺失时直接返回明确环境提示，避免只显示通用进程启动失败。
- 转发规则入口地址优先使用入口组域名，并在域名缺失时回退到组成员的真实入口地址；地址暂不可用时显示明确状态，不再把资源名称当作可复制入口。

### 修复

- 修复出口组调整第一台主机后，GOST/Nginx 规则业务端口仍按旧出口序号映射的问题；规则端口现在按主机迁移并执行并发安全的端口预留。
- 修复出口策略为“不使用”时，GOST 仍可能在后续出口创建空闲探测监听的问题；探测拓扑指纹现在包含出口策略，策略切换后不会继续接受旧拓扑结果。

- 修复 ForwardX 自定义加密隧道在 TCP 连接出现正常半关闭后仍按 30 秒兜底关闭另一方向的问题，避免直播/长响应流在服务端继续下发数据时被过早断开。
